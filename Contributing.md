# WasmPulse Contributing Guide

Thank you for interest in the WasmPulse project! Please follow the guidelines below to ensure smooth sailing.

## Issues

For bug reports, feature suggestions, or code contributions, create a new issue or start a discussion. 
PR's are welcome, but have to have a rich description for maintainers to understand and review it witout uncertainty.

## Pull Requests

- Create a new branch for each PR.
- All Git commits need to be signed and marked as "Verified": signed with a GPG, SSH, or S/MIME that is successfully verified by GitHub.
- Commit messages should be simple and descriptive.
- Adjust the contribution according to PR reviewer's comments.
